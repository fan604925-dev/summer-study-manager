# 2026-07-24 上午 - 再次修复家务积分仍进学习积分池（v19）

## 现象
- 用户截图：哥哥学习积分 77，哥哥家务积分 0；奖励页同时出现两个标题「可选庆祝方式」和「可选庆祝方式（用学习积分）」
- 判断：用户手机仍运行旧缓存页面，且 v18 的 `_repairData` 对部分旧数据未正确同步 `cashable` 标记

## 修复内容
- `rewards.html`：删除第 64 行重复的旧标题「可选庆祝方式」，只保留 JS 渲染的「可选庆祝方式（用学习积分）」
- `js/store.js`：
  - `DATA_VERSION` 升 19，强制所有用户重新触发 `_repairData`
  - 增强 `_repairData` 中实例 `cashable` 同步逻辑：模板存在时按模板；模板不存在时按实例名称/任务名兜底识别 KNOWN_CHORE_NAMES
- `service-worker.js`：`CACHE_NAME` 从 `ssm-v6` 升级到 `ssm-v7`，激活时自动清理旧缓存

## 测试
- `test_store.js` 全部 61 项通过（含新增：自定义洗碗模板识别 cashable、自定义洗碗实例同步 cashable 标记、完成后家务积分增加）

## 部署

# 2026-07-24 上午 - 支持自定义家务任务（v20）

## 现象
- 用户反馈：哥哥默认"洗碗"已正确计入家务积分，但弟弟手动添加的"整理书桌和沙发"仍计入学习积分
- 同时用户手机仍显示两个"可选庆祝方式"标题，说明旧缓存未更新

## 修复内容
- `js/store.js`：
  - `DATA_VERSION` 升 20，强制触发 `_repairData`
  - 扩展 `KNOWN_CHORE_NAMES`：新增「整理、擦桌子、叠衣服、倒垃圾、浇花、晾衣服、收衣服」等常见家务关键词
- `tasks.html`：
  - 添加任务弹窗增加 💰"计为家务积分（可换零花钱）"开关
  - 任务详情弹窗显示当前积分类型（学习/家务）
  - 自定义任务可在详情中一键切换学习积分 / 家务积分
  - 完成提示区分 💰家务积分 / ⭐学习积分
- `service-worker.js`：`CACHE_NAME` 升级到 `ssm-v8`，强制清理旧缓存

## 测试
- `test_store.js` 全部 61 项通过

## 部署
- commit `d1c71d9`，push main 成功
- GitHub Pages 重建后验证：`DATA_VERSION='20'`，`rewards.html` 重复标题已消失

## 用户操作
- 必须点首页「🛠️ 一键修复」清除旧缓存，加载 v20 后：
  1. "整理书桌和沙发"会自动被识别为家务任务（名字含"整理"）
  2. 若以后添加其他特殊家务，可在添加时手动勾选"计为家务积分"，或在任务详情中切换
