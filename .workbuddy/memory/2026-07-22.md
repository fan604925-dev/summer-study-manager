## 2026-07-22 第三次修复（用户反馈"还是看不到任务、倒计时显示已过"）
**根因确认**：浏览器 Service Worker 一直缓存着旧版 `store.js`/`index.html`，用户刷新后仍运行旧代码（旧版 examDate 是过去日期导致"已过"，旧任务模板缺失导致空白）。这是"刷新不更新"的核心痛点。

**根治方案（彻底解决缓存 + 数据初始化）**：
1. Service Worker 改为**完全网络优先**（HTML/JS/CSS/manifest 都走网络，仅离线回退缓存），CACHE_NAME 升到 `ssm-v6`，不再缓存 JS/CSS。图标仍缓存优先。
   - 效果：每次刷新都拿最新代码，彻底消除"刷新后还是旧界面"。
2. 数据版本升到 `DATA_VERSION='12'`，`_repairData` 强化：模板数量 < 默认40% 或为空时**强制重新填充**默认模板；奖励为空时强制重新填充。避免旧缓存数据导致空白。
3. 倒计时健壮化：`getExamCountdown` 在 examDate 为空返回 `null`；UI（index/sprint）区分"未设置(∞)/已过期(0)/正常(天数)"三种状态，不再误显"已过"。
4. **考试日期可关闭**：store 新增 `setExamDate()`；设置页新增考试日期输入框 + "关闭倒计时"按钮。满足用户"去掉截止时间以后一直用"的长期需求。
5. 首页新增"🛠️ 一键修复"按钮（调用 `app.clearCacheAndReload()`），卡住时一键清理 SW 缓存并重载，不删数据。
6. 验证：`test_store.js` 全部通过（倒计时33天、关闭返回null、模板损坏自动恢复、自定义任务持久化、旧积分迁移）；6页面语法 OK。

**给用户的关键操作指引**：
- 刷新（Ctrl+Shift+R）后应看到最新界面。
- 若仍旧：首页点「🛠️ 一键修复」或设置页「清除缓存并刷新」。
- 若彻底卡死：DevTools → Application → Service Workers → Unregister + Clear Storage，再硬刷。
